Skill设计原则

  • 借用现有平台能力，不重复造轮子
  • 关注代码搜索不准确的信息（表血缘/调度依赖等）
  • 按需加载：metadata常驻 → body按需 → resources使用时
  • 可组合性：先查依赖 → 再找脚本 → 再读代码

  • architecture/ — 数据流向、组件关系

访问：MCP → Mem0 API（向量检索）

写入时机

  • 需求完成后：记录需求、方案、反馈
  • 遇到问题后：记录故障原因和解决方案
  • 业务验收后：记录偏好和反馈意见

读取时机

  • 新需求：检索历史相似需求经验
  • 架构设计：参考类似技术方案
  • 代码生成：复用已验证代码模式

维度2：AI幻觉四层防御体系 全自动化

Layer 1: 输入层防御 Wiki RAG增强 · CLAUDE.md规范约束 · Skill工具代替自由生成 减少60-80%
Layer 2: 生成层防御 规范模板约束 · Few-shot示例引导 · 强制引用来源 —
Layer 3: 输出层防御 SQL语法检查 · 单元测试验证 · 依赖项验证 精度89.5%
Layer 4: 审查层防御 AI自动扫描 · Human代码审查 · 运行时监控 组合96%+
Loop内自我纠错
  • SQL语法检查 → 解析错误 → 重新生成
  • JSON格式验证 → 补全缺失字段
  • 依赖合法性 → Skill查询确认
  • 规范一致性 → 对照CLAUDE.md自动重命名
  • 完整性检查 → 识别缺失 → 补充生成
防御在Loop中的位置
  • Layer 1 → 所有阶段的输入准备环节
  • Layer 2 → 阶段1-4的Agent生成环节
  • Layer 3 → 阶段4自检 + 阶段5 CI/CD
  • Layer 4 → CP1-CP6 六大检查点
