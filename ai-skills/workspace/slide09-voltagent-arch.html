VoltAgent 原理一: Agent .md 结构与加载机制

Agent 文件标准结构

--- YAML Frontmatter ---

name: api-designer

description: "Use when designing APIs"

tools: Read, Write, Edit, Bash, Glob, Grep

model: sonnet

---

# 内容模板 (7 层)

1. 角色定义 (senior + expertise)

2. When invoked: 初始化 4 步

3. 核心能力 Checklist (7-8 项)

4. 专业领域 (8-12 子项/节)

5. Communication Protocol (JSON)

6. Development Workflow (3 Phase)

7. Integration with other agents

工具权限按角色分配

审核员: Read, Grep, Glob (只读)

研究型: + WebFetch, WebSearch

开发者: Read, Write, Edit, Bash, Glob, Grep

模型选择策略

opus: 最复杂 (code-reviewer 多语言分析)

sonnet: 标准 (api-designer/docker-expert)

haiku: 简单高频 (task-distributor)

Claude Code 加载流程

Scan: 扫描 ~/.claude/agents/ + .claude/agents/

|

Parse: 解析每个 .md 的 YAML Frontmatter

|

Dedup: 项目级覆盖全局级 (同名去重)

|

Index: 构建搜索索引 (name/desc/category)

|

Match: 用户消息 -> 匹配 description 关键词

|

Load: 解析 .md 正文 -> 生成 System Prompt

|

Filter: 应用 tools 字段 -> 工具权限沙箱

System Prompt 生成

.md 正文 -> 角色定义 + When invoked 步骤

+ 能力清单 + 专业领域 + 通信协议

通常 2000-4000 tokens，完整角色定义
