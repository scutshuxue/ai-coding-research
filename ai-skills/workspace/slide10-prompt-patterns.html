VoltAgent 原理二: Prompt Engineering 模式分析

代理 Prompt 模式对比

api-designer (sonnet)

"senior API designer specializing in..."

When invoked:

1. Query context -> existing APIs

2. Review -> current endpoints

3. Analyze -> requirements

4. Implement -> API design

docker-expert (sonnet)

"senior containerization specialist..."

Checklist: image_size > 70% reduced

build_time > 60% reduced

security_score > A rating

code-reviewer (opus)

"senior code reviewer" | 最复杂任务

Output: severity-ranked findings

critical > warning > info

标准化输出格式

Progress Tracking (所有代理)

{"agent": "api-designer",

"status": "designing endpoints",

"progress": {"endpoints": 12}}

Delivery Notification

"[API] 45 endpoints, OpenAPI 3.1"

"[Docker] 847MB->89MB (89% cut)"

"[Review] 47 files, 2 critical"

共同 Prompt 模式

"senior" 前缀 -> 建立权威性

"expertise in" -> 知识边界定义

When invoked 4 步: 信息->分析->优化->交付

Communication Protocol: JSON 标准化

Integration: 声明协作代理
